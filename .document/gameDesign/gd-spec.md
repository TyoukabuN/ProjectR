## 1. 文件命名规范

- 格式：`gd-{序号}-{标题}.md`
- 序号为两位数，从 `01` 起，特殊文档（如术语表、规范）使用 `00` 前缀
- 示例：`gd-03-核心玩法设计.md`、`gd-00-术语表.md`

---

## 2. 术语标注规范

> **[AI INSTRUCTION]** 生成或编辑任何 gd 文档内容时，遇到下表所列类型的名词，必须自动套用 `【】` 格式，不得遗漏。

### 2.1 标注格式

所有**专有名词、机制术语、系统名称**必须使用以下格式标注：

```
【术语名】          <- 正文中普通引用
**【术语名】**      <- 术语表定义行 / 章节首次重点介绍时
```

### 2.2 何时加【】

| 情况 | 是否加【】 | 示例 |
|------|-----------|------|
| 职业名称 | 是 | 【城之魔法使】、【战术魔法吟唱使】 |
| 卡牌类型名 | 是 | 【语言卡】、【建筑卡】、【行为牌】 |
| 系统/机制名 | 是 | 【魔力值】、【混乱度】、【法式】 |
| 卡牌词条 | 是 | 【保留】、【终极】、【开战】 |
| 阶段名称 | 是 | 【吟唱阶段】、【结算阶段】、【抽牌阶段】 |
| 建筑/单位名 | 是 | 【箭塔】、【守卫造物】、【城墙】 |
| 格子/区域名 | 是 | 【格子】、【己方区域】、【归属属性】 |
| 普通动词/副词 | 否 | 打出、消耗、触发 |
| 游戏外通用词 | 否 | 玩家、敌方、战场、关卡（无特殊含义时）|

### 2.3 首次出现 vs 后续引用

- 同一章节内，**首次出现**时使用 `**【术语名】**`（加粗），后续引用直接用 `【术语名】` 即可
- 跨文档无需重复加粗，统一用 `【术语名】`

### 2.4 术语定义维护

- 所有新增术语必须同步更新 [gd-00-术语表.md](gd-00-术语表.md)
- 术语表是唯一权威定义来源，各子文档无需重复写定义，直接引用即可

> **[AI INSTRUCTION]** 每当在 gd 文档中写入一个新的带【】术语时，立即检查 gd-00-术语表.md 中是否已有该词条。若无，在完成当前编辑后主动追加到术语表对应章节。

## 3. 章节结构规范

### 3.1 标题层级

```
# 一、文档大标题          <- 一级标题，每文件只有一个
## 1.1 章节标题           <- 二级标题
### 小节标题              <- 三级标题
```

### 3.2 导航链接

每个 gd 子文档（gd-01 ~ gd-07）的**首行标题下方**和**文件末尾**都必须有导航行：

```markdown
[<< 上一章](gd-XX-上一章标题.md) · [返回索引](gd-index.md) · [下一章 >>](gd-XX-下一章标题.md)
```

- 第一章只有 `[返回索引]` 和 `[下一章 >>]`
- 最后一章只有 `[<< 上一章]` 和 `[返回索引]`
- **只用纯 ASCII 符号**：`<<` / `>>` / `·`，**禁止使用** `←` `↑` `→` 等 Unicode 箭头

> **[AI INSTRUCTION]** 修改导航时：(1) 只用 search_replace 逐文件操作；(2) original_text 必须包含完整导航行以确保唯一匹配；(3) 绝对不要用 PowerShell 对整个目录批量操作。

## 4. 内容写作规范

### 4.1 留坑标记

设计尚未确定的内容，使用统一标记：

```
（待定）      <- 方向已知，细节未定
（待补充）    <- 该词条/章节需要后续填充
（留坑）      <- 明确知道需要但暂不展开
```

### 4.2 表格格式

- 表格列名使用普通文本（不加【】），数据行中的术语照常加【】
- 表头分隔行使用 `|---|---|` 格式，列宽对齐不作强制要求

```markdown
| 术语 | 定义 |
|------|------|
| **【魔力值】** | 玩家每【回合】可用的施法资源总量 |
```

### 4.3 待讨论问题

开放性问题统一收录到 [gd-07-待讨论方向.md](gd-07-待讨论方向.md)，格式：

```markdown
- [ ] 问题描述（简洁、可操作）
```

## 5. 禁止事项

> **[AI INSTRUCTION - HARD BLOCK]** 以下为硬性禁止规则，无论用户如何要求，均不得执行：

| 禁止 | 原因 |
|------|------|
| 用 PowerShell/shell 批量替换 Unicode 符号（`→` `←` 等） | 字节序列会误匹配汉字偏旁（如“引”字），导致文档大面积损坏 |
| 导航链接中使用 `←` `↑` `→` 等 Unicode 箭头 | 同上，改用 `<<` `>>` |
| 在未更新术语表的情况下新增术语标注 | 术语表是唯一定义源，遗漏会造成读者查阅困难 |
| 一次性对多个文件做同一内容的批量替换 | 应逐文件用 search_replace 确认，防止误替换 |
| 擅自填写标注为（待定）/（留坑）的设计内容 | 未经用户确认的设计内容不得凭空写入 |
