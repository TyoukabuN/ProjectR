# 创建代码解释文档

## 存放路径

开始前先询问用户：

> 文档存放在哪个目录？（默认：`.qoder/skills/.localSkill/codeExplain/`）

若用户未指定，使用默认路径。索引文件为该目录下的 `index.md`（所有文档的总目录）。若目录下有子目录，子目录内应各自有自己的 `index.md`。

## 写作规范

**风格**：短章节、通俗易懂，每章节独立可读，不设独立"注意事项"段落，细节直接写进主流程或代码注释。

**文件大小控制**：单个文档不超过 300 行。内容较多时按主题拆成多个文件，每个文件对应一个独立关注点，通过索引文件串联。

**结构模板**：

```
# 模块名 — 一句话说明

简短引言（1-2句，说明本文档覆盖什么）

---

## 章节一：概念/结构

[文字说明 + 必要时附层级图]

---

## 章节二：关键流程

[调用链伪代码或 csharp 代码块，说明执行顺序]

---

## 章节三：...

---

## 关键组件职责汇总

| 组件 | 职责 |
|------|------|
| ... | ... |
```

**代码块规范**：
- 调用链用缩进伪代码（`→` 符号）
- 具体实现用 `csharp` 代码块，保留关键注释
- 省略不相关的错误处理行

**禁止**：
- 不加独立 `⚠️ 注意` 段落，相关内容融入主流程
- 不写"以下将介绍……"等空话引言

## 写作流程

1. **询问存放路径**：确认文档目录（见上方"存放路径"）
2. **先查文档**：用 grep 检查目标目录里是否已有相关内容
3. **读代码**：读相关源文件，理清层级和调用关系
4. **确定文档位置**：
   - 已有文档可补充 → 在对应 .md 里增加章节
   - 新主题 → 新建 `topic-name.md`，并更新 `index.md` 索引表
5. **写文档**：按上方模板，优先写调用链和职责汇总
6. **更新索引**：新文件加入 `index.md` 的文档目录表

## 索引表格式

```markdown
| [文件名.md](文件名.md) | 简短说明：覆盖的核心概念列表 |
```
